# Stablecoin Vulnerability Writeup

Discovery, demonstration, and fix of a critical vulnerability in a
widely-audited stablecoin smart contract.

## Summary

- Found a critical vulnerability in a production stablecoin contract
- Demonstrated the exploit in a controlled environment
- Provided a complete fix
- Competed against a senior security team and won

## Vulnerability Class

- Access Control Bypass
- Logic Error in collateral calculation

## Timeline

- Discovery — Independent security research
- Demonstration — Controlled exploit proof
- Fix — Patch proposed and validated
- Disclosure — Responsible disclosure process followed

## Impact

- Potential loss of user funds
- Collateral manipulation
- Protocol insolvency risk

## Key Takeaway

Even heavily audited contracts can contain critical vulnerabilities.
Multiple audit rounds do not guarantee security.
